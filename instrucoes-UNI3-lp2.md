Neste  projeto,  propõe-se  desenvolver  um   sistema  para  gerenciar  empréstimos  e  devoluções  em  uma
biblioteca escolar. A aplicação deve controlar o acervo de livros e as transações de empréstimo, mantendo
todas   as   informações   organizadas   para   biblioteca   e   usuários.   Cada   biblioteca   possui   controle
independente   do   seu   acervo   de   livros   e   empréstimos.  Maiores   informações   considerando-se
especificações divulgadas por docente do componente curricular de Estruturas de Dados.

Neste projeto, propõe-se desenvolver um sistema para gerenciar aulas em uma academia de ginástica.
Cada instrutor pode oferecer várias modalidades de aulas (como yoga, musculação, etc.), e os alunos se
inscrevem nas aulas de sua escolha. A aplicação deve controlar a programação de aulas e as inscrições
dos alunos. Cada instrutor possui controle independente das aulas que ministra.  Maiores informações
considerando-se especificações divulgadas por docente do componente curricular de Estruturas de
Dados.

Neste projeto, propõe-se criar um sistema para gerenciar reservas de quartos em um hotel. O sistema
deve   controlar   o   inventário   de   quartos,   as   reservas   efetuadas   por   hóspedes   e   a   ocupação   das
acomodações.   Cada   hotel   possui   um   controle   independente   de   seus   quartos   e   reservas.  Maiores
informações considerando-se especificações divulgadas por docente do componente curricular de
Estruturas de Dados.

Neste projeto, propõe-se desenvolver um sistema para gerenciar reservas de passagens interestaduais por
ônibus. Considerando linhas de ônibus com horários definidos, cada viagem possui assentos limitados. A
aplicação   deve   controlar   horários,   assentos   e   reservas   dos   passageiros.   Cada   linha   de   ônibus   possui
horários   e   rota   específicos.  Maiores   informações   considerando-se   especificações   divulgadas   por
docente do componente curricular de Estruturas de Dados.

Neste projeto, propõe-se desenvolver um sistema para gerenciar projetos em uma empresa de TI. Cada
projeto reúne um conjunto de tarefas e membros de equipe responsáveis por executá-las. A aplicação
deve manter o controle das informações de tarefas e alocação de equipe. Cada projeto possui um controle
independente   de   suas   tarefas   e   prazos.  Maiores   informações   considerando-se   especificações
divulgadas por docente do componente curricular de Estruturas de Dados.

Neste projeto, propõe-se desenvolver um sistema para gerenciar competições esportivas amadoras. Cada
equipe participa de um campeonato com várias rodadas de jogos. A aplicação deve manter o controle da
programação de partidas, resultados e classificação. Cada equipe possui um controle independente de sua
agenda de jogos e pontuação.  Maiores informações considerando-se especificações divulgadas por
docente do componente curricular de Estruturas de Dados.

Neste projeto, considere a existência de uma locadora de veículos. O projeto consiste no desenvolvimento
de aplicação para cadastro, a qualquer momento, de novo veículo, exigindo-se, para tal, entrada dos
seguintes dados: a) placa; b) modelo; c) ano; d) cor predominante; e e) quilometragem. O uso de coleções
genéricas concretas para a manipulação da lista de veículos é obrigatório. Além disso, a  aplicação
deve dispor ainda das seguintes funcionalidades (a serem acessadas a qualquer momento) e/ou restrições,
a saber:

•
•

Impedimento de existência de 2 (dois) ou mais veículos cadastrados com a mesma placa;
Registro de locação de veículo a partir de entrada de respectiva placa e de nome do cliente;
Impedimento de existência de 2 (dois) ou mais locações simultâneas para um mesmo veículo;
Registro de entrega de veículo locado a partir da entrada de respectiva placa e de quantidade de

modelo);
Consulta   de   veículos   cadastrados   de   determinada   cor   predominante   (para   fazê-lo,   é   necessário
informar tal cor);
Consulta de veículos cadastrados cuja quilometragem atual esteja em determinada faixa (para fazê-lo,
é necessário informar quilometragem mínima e quilometragem máxima);
Listagem de dados de veículos que se encontram até então locados acompanhados de nomes de
respectivos clientes;
Listagem de dados de veículos que não se encontram até então locados.

Tema 8 (Sócios de
Clube)

Neste projeto, considere associados aos sócios de determinado clube. Para cada sócio são mantidas as
seguintes informações: número (sócio), nome, CPF, idade, profissão, endereço e data de inscrição no
clube. Além disso, há registro de dados de pagamento das cotas anuais de cada sócio. O projeto consiste
no desenvolvimento de aplicação para cadastro, a qualquer momento, de novo sócio. O uso de coleções
genéricas concretas para a manipulação da lista de sócios é obrigatório. Além disso, a aplicação deve
dispor ainda das seguintes funcionalidades (a serem acessadas a qualquer momento) e/ou restrições, a
saber:

•
•

Geração automática de número de sócio, quando da efetivação de novo cadastro;

Registro   de   pagamento   de   determinado   cota   anual   de   determinado   sócio   a   partir   de   informe   de
número de sócio, valor e ano de referência;
Impedimento de 2 (dois) ou mais registros de pagamento de cota anual de mesmo sócio se referentes

Listagem de sócios que satisfazem determinado critério de pesquisa (iniciais de nome ou profissão);

Alteração de dados de determinado sócio excetuando-se número de inscrição;
Listagem de dados de sócios por ordem alfabética de nome;
Listagem de dados de sócios por ordem decrescente de data de inscrição.
